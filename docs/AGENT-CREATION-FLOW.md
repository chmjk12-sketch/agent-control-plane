# Agent 智能创建流程 V3

> 日期: 2026-06-15
> 版本: v3.0 (TRAE 智能驱动)

---

## 一、核心理念变化

| 维度 | V2 (旧) | V3 (新) |
|------|---------|---------|
| 起点 | 用户手动填表 | TRAE 理解需求 → 确认方案 |
| UI 选择 | 从 3 个固定模板选 | LLM 搜索同类顶尖产品 → 筛选 → 用户选择 |
| 规范传递 | 模板仓库 .trae/rules.md | TRAE 内置规范 + 模板仓库双重保障 |
| 用户参与 | 每步都操作 | 仅 3 次重大确认 |
| 自动化程度 | 半自动 | 尽量智能，尽量少参与 |

## 三、用户参与度对比

| 步骤 | 用户操作 | 自动完成 | 用户决策 |
|------|----------|----------|----------|
| 1. 需求理解 | 描述需求 | TRAE 解析意图、推断技术栈 | **确认需求** |
| 2. UI 搜索 | 选择 UI 风格 | LLM 搜索、筛选、评分 | **确认 UI** |
| 3. 方案生成 | 无 | TRAE 生成全部代码 | 无 |
| 4. 本地验收 | 查看结果 | 自动启动、自动测试 | 确认验收 |
| 5. 推送部署 | 无 | Git push + CI/CD | 无 |
| 6. 地址确认 | 选择访问方式 | 自动配置路由 | **确认地址** |
| 7. 完成 | 无 | Caddy + 通知 + 监控 | 无 |

**用户只需 3 次重大确认：需求、UI 风格、访问地址。**

## 四、UI 智能搜索详细机制

### 4.1 搜索流程

```
TRAE 接收到 Agent 类型 + 需求描述
  │
  ▼
LLM 生成搜索关键词
  │  例: "股票分析" →
  │    英文: "stock analysis dashboard", "financial chart UI", "trading platform"
  │    中文: "股票分析看板", "金融数据可视化", "量化交易界面"
  │
  ▼
并行搜索多个来源
  │
  ├─→ GitHub: 搜索 trending repos (按 star/更新时间排序)
  │     例: akshare-demo (3.2K stars), ta-lib-python (9K stars)
  │
  ├─→ Dribbble/Behance: 搜索金融类 Dashboard 设计
  │     例: "Financial Dashboard UI Kit", "Stock Trading Interface"
  │

  │     例: TradingView (专业K线), 同花顺 (简洁中文), 富途牛牛 (移动端)

  │
  ├─→ 功能匹配度 (0-100): 是否有 K线、行情、报告

  │
  ▼
```

## 五、TRAE 内置规范 (不依赖模板仓库)

除了模板仓库的 `.trae/rules.md`，TRAE 本身也内置了 Agent 规范：

```
TRAE 内置规范 (始终生效)
  │
  ├─→ 技术栈约束
  │     ├─→ 后端必须: Python 3.11 + FastAPI + Pydantic v2
  │     ├─→ 前端可选: React 18 + TypeScript + Tailwind CSS
  │     └─→ AI 调用: OpenAI SDK (兼容 DeepSeek/Claude)
  │
  ├─→ 强制端点
  │     ├─→ GET /health → {"status": "ok"}
  │     ├─→ GET /metrics → Prometheus 格式
  │     └─→ POST /api/v1/{action} → 业务 API
  │
  ├─→ 强制文件
  │     ├─→ agent.yaml (元数据)
  │     ├─→ Dockerfile (构建)
  │     ├─→ docker-compose.yml (本地开发)
  │     ├─→ tests/ (Mock 测试)
  │     └─→ .github/workflows/deploy.yml (CI/CD)
  │
  ├─→ 控制平面集成
  │     ├─→ 启动时注册
  │     └─→ 请求上报 (异步)
  │
  └─→ 测试要求
        ├─→ 100% Mock 外部调用
        └─→ CI/CD 无需真实 API Key

模板仓库 .trae/rules.md (项目级补充)
  │
  └─→ 项目特定的业务规范
        ├─→ 领域特定的数据模型
        ├─→ 领域特定的 API 设计
        └─→ 领域特定的测试用例
```
